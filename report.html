Project report Project goals

The objective of the project is to study the populations of the different types of penguins living in Antarctica.

Data Source

The data used is the list of all penguin observations in Antarctica (from various sources) compiled by the Mapping Application for Penguin Populations and Projected Dynamics (MAPPPD). For each observation in the dataset, the following data is given:

  • The name given to the observation site, as well as the site initials
  • The coordinates of the site
  • The common name given to the species of penguin
  • The number of penguins observed
  • The accuracy level of the observation (1 being the most accurate, 5 the least)
  • The exact date of the observation
  • The year of the beginning of the season (For example, January 2010 observations belong to a season starting in 2009)
  • The type of penguin observed (Adults, chicks, nests)
  • The source of the observation
Other data is also available but was not used for this project.

Features

Our project is set up as an interactive website, allowing the user to discover the dataset and its implications. Using a menu at the top of each page, the user can browse different sections of the site.
The index of the site is a homepage, allowing access to all the pages.
The map page contains our interactive map, allowing the user to see the locations and species of all observed penguins for any given years, as well as a heatmap of penguin activity.
The penguin page displays a chart of all observations, grouped by month, allowing the user to see how the data is distributed as well as see, for any given parameters, the observations with the most penguins corresponding to those parameters.
The final page is the one you're reading right now!

Technologies

The website is based on a free to use Bootstrap template from StartBootstrap. Using a base template, we can quickly start to add our own content without worrying about the basics.
The graphs were creating using D3.js, dc.js and Crossfilter. Using Crossfilter, we can select what data dimensions to consider and natively feed to DC.js graphs, which in turn uses D3.js to render and display them. Crossfilter allows interactive data filtering without completely recreating all graphs, allowing for fast modification of the data displayed. TODO interactive map
